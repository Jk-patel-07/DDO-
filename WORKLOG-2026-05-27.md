# DDO App Work Note

Date: 2026-05-27

## App stability
- Fixed the `RightTray.jsx` white-screen/runtime issue caused by `requestBackendJson` being used before initialization.
- Moved backend request handling into a safe top-level helper.
- Added a runtime fallback boundary so the app shows a recovery UI instead of a blank white screen if a component crashes.
- Kept the status bar visible more reliably so the app does not appear blank on load.

## Security panel
- Added a `Security Check` option inside the App Settings popup.
- Connected the Security Check panel to a real backend route: `/api/security/status`.
- Showed live status for:
  - File Upload Protection
  - Link Protection
  - Login Protection
  - API Key Protection
- Added loading and unavailable-state handling.

## Login and registration
- Kept the existing login popup design.
- Added a `Register here` flow that opens a separate registration popup.
- Added registration fields:
  - Email ID
  - First Name
  - Middle Name
  - Last Name
  - More Information
  - Phone Number
  - Password
  - Confirm Password
- Added frontend validation for email, names, password strength, password match, and optional phone number.
- Added `Continue with Google` login option in the auth flow.

## Backend auth/register
- Extended the backend auth system to support registration and login.
- Added secure password hashing on the backend.
- Added support for local registered users plus existing auth session handling.
- Added `.ddo-users.json` to `.gitignore`.

## Register button/backend connection
- Fixed the Register button to call the backend using the full URL:
  - `http://127.0.0.1:5000/api/auth/register`
- Updated the payload to match backend expectations.
- Removed robot verification from the registration flow.
- Improved register error handling so it shows real backend messages instead of only `Failed to fetch`.
